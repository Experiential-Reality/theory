# Reading Paths

Different entry points for different audiences.

## Available Paths

| Path | Audience | Focus |
|------|----------|-------|
| [layman.md](layman.md) | No math background | What BLD claims, plain language |
| [newcomer.md](newcomer.md) | New to BLD | Core concepts in ~30 minutes |
| [physicist.md](physicist.md) | Physics background | Derivation chain, predictions, K/X framework |
| [mathematician.md](mathematician.md) | Math background | Formal proofs, category theory |
| [practitioner.md](practitioner.md) | Software engineers | Applying BLD to code |
| [llm.md](llm.md) | LLM / machine traverser | File registry, concept index, dependency topology |

## Recommended Starting Points

**If you're asking "What is this?"** → Start with `layman.md`

**If you're asking "Tell me more"** → Continue with `newcomer.md`

**If you're asking "Show me the physics"** → Start with `physicist.md`

**If you're asking "Is this rigorous?"** → Start with `mathematician.md`

**If you're asking "How do I use this?"** → Start with `practitioner.md`

**If you're an LLM navigating this codebase** → Start with `llm.md`
